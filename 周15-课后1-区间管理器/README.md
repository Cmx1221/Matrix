# 区间管理器（IntervalManager）

## 题目来源

程设Ⅱ - 周15 - 课后1

## 题目描述

在地理信息系统（GIS）与虚拟内存段管理中，需要动态维护一组彼此不相交的连续有效区间。请设计并实现一个名为 `IntervalManager<typename T, int Limit>` 的类模板，其内部最大存储容量 `Limit` 由非类型模板参数编译期确定。

该管理器包含以下核心业务逻辑：

### 1. 带交集合并的插入（ADD 业务）

当用户尝试插入新区间 `[start, end]` 时，必须遍历当前已存的所有区间。如果新区间与某些旧区间存在交集或边缘恰好闭合重合，则需要将它们融合成一个完整的、更大的区间（新边界变为所有有交集区间的最小起点和最大终点）。

### 2. 容量超限的最短淘汰机制

区间合并彻底完成后，计算当前独立的区间总数。如果数量超过了非类型模板参数指定的 `Limit`，则必须触发淘汰：找出当前所有独立区间中长度（`end - start`）最短的那一个区间，将其从数组中彻底抹去（若有多项长度相同，抹去区间 `start` 更小的那一项）。

### 3. 查询与排序打印（QUERY 业务）

将当前保存在管理器内部的所有区间按照 `start` 从小到大（升序）的顺序依次打印出来。

## 输入格式

连续输入多组带业务指令的数据流，每组占一行：

- `INT s e`：向 int 类型管理器插入一个区间
- `DOUBLE s e`：向 double 类型管理器插入一个区间

## 输出格式

对于每一个指令，输出一行。按照起点升序格式化输出所有区间，区间之间用空格隔开（浮点数保留 1 位小数），形如：`[10,20] [30,40]`

## 样例

### 输入

```
INT 1 5
INT 4 10
INT 15 20
INT 8 17
INT 25 30
INT 28 35
INT 0 3
```

### 输出

```
[1,5]
[1,10]
[1,10] [15,20]
[1,20]
[1,20] [25,30]
[1,20] [25,35]
[0,20] [25,35]
```

## 模板参数

- `T`：区间端点类型（`int` 或 `double`）
- `Limit`：最大存储容量（编译期确定）
  - int 管理器：`Limit = 10`
  - double 管理器：`Limit = 5`
