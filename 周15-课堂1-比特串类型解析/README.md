# 比特串类型解析

## 题目来源

程设Ⅱ - 周15 - 课堂1

## 题目描述

在分布式通信与硬件接口开发中，底层数据通常以统一的二进制比特流进行传递。为了测试流处理芯片的低位分流算法，系统需要根据不同的目标转换类型，动态截取并重构内存块：

- 如果目标是 **32 位类型**（`int` 或 `float`）：算法将接收长度为 32 位的二进制比特字符串。
- 如果目标是 **64 位类型**（`double`）：算法将接收长度为 64 位的二进制比特字符串。

实现 `template<typename T> T decode_bits(const string& bits)` 模板函数，根据接收的比特字符串，还原为目标模板类型 T 的实际数值并返回。

### 内存布局与标准说明

- **底层编码**：所有浮点数类型（`float` 和 `double`）在二进制层面均严格遵循 **IEEE 754** 标准编码。整数类型为有符号整数。
- **比特串规定**：接收的比特串是**大端序**打包的。即 `bits[0]` 代表编码最高位比特，`bits[1]` 是次高位，以此类推——比特串索引顺序与编码位地址顺序相反。

## 输入格式

多组测试数据，每组占一行，包含两个字段：

- 第一个字段：目标转换类型（`INT`、`FLOAT` 或 `DOUBLE`）
- 第二个字段：纯二进制比特串（`INT`/`FLOAT` 时为 32 位，`DOUBLE` 时为 64 位）

## 输出格式

每组测试数据输出一行，为还原后的目标类型数值。浮点数保留 **4 位小数**。

## 样例

### 输入

```
INT 00000000000000000000000000101010
FLOAT 00111111100000000000000000000000
DOUBLE 1011111111110000000000000000000000000000000000000000000000000000
```

### 输出

```
42
1.0000
-1.0000
```
