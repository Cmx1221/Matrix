### 第1题 - 单选题

- **A.** 宏定义替换

- **D.** 对运算符优先级的修改

- **C.** 运算符重载不能改变运算符的操作数个数

### 第13题 - 单选题

为什么前置自增（++a）通常返回左值引用（T&），而后置自增（a++）通常返回右值副本（T）？

- **A.** 因为 C++ 编译器强制要求必须这么写，否则会报语法错误。
- **B.** 前置运算需要返回修改后的对象本身；后置运算需要"先用后增"，必须返回一份自增前的历史状态副本。
- **C.** 返回引用会占用更多的内存空间，后置自增为了优化性能所以返回值。
- **D.** 两者其实都可以返回引用，只是习惯问题。

> **标准答案：B**

- **B.** 移动赋值（Move Assignment）

- **D.** 零拷贝（Zero Copy）

### 第15题 - 单选题

当类的成员变量包含指针（并在堆上动态分配了内存）时，如果使用缺省的赋值运算符（浅拷贝）执行 a = b;，会导致什么最严重的后果？

- **A.** 编译无法通过，提示禁止给指针赋值。
- **B.** a 和 b 的指针内容会各自独立，互不影响。
- **C.** 会导致 a 原有的内存泄漏，且析构时同一块内存会被释放两次（Double Free），引发段错误。

### 第17题 - 单选题

根据标准规范，实现"深拷贝"赋值运算符时，操作指针成员的正确步骤顺序是什么（假设非自赋值且空间需重分配）？

- **A.** 先分配新存储 new int[]，再 delete[] 旧存储，最后执行复制。
- **B.** 先执行数据复制，再 delete[] 原有存储。
- **C.** 直接通过 new int[] 重新覆盖原指针，不进行 delete 操作以提高性能。

### 第19题 - 单选题

在实现复合运算赋值（如 +=）时，课件提到了一种实现思路：先调用已重载的加法（operator+），再将结果赋给自身（*this = this->operator+(other);）。这种方法的特点是？

- **A.** 它比直接修改成员变量（如 x += other.x）更加高效，是唯一推荐做法。
- **B.** 虽然代码得到了复用，但会因为产生临时对象和额外的拷贝赋值导致效率低于直接修改自身成员。
- **C.** 它是实现复合赋值唯一正确的方法。
- **D.** 这种写法在语法上是不被 C++ 编译器允许的。

> **标准答案：B**

- **A.** a.a 和 b.a 指向同一个内存地址（共用一处堆空间）。
